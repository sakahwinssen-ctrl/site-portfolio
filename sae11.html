SAÉ 11

Lot 3 : Découverte réseau

Cette SAÉ m’a permis de travailler sur la découverte d’un réseau avec Nmap : repérer les machines actives, identifier les systèmes, les services ouverts et comprendre les précautions à prendre avant de lancer un scan.

Retour aux SAÉ Voir le diaporama

Présentation de la SAÉ

Avec mon groupe, nous avons travaillé sur le lot 3 : Découverte. Le but était d’expliquer comment analyser un réseau avec Nmap.

Notre diaporama présente plusieurs étapes :

  • repérer les machines actives du réseau avec nmap -sn ;
  • identifier le système d’exploitation avec nmap -O ;
  • voir les services ouverts avec nmap -sV ;
  • obtenir plus de détails sur les versions avec --version-all.

Nous avons aussi expliqué qu’un scan réseau doit être fait avec autorisation, car il peut être considéré comme intrusif ou gêner le fonctionnement du réseau.

Mes tâches et mes preuves

J’ai participé à la préparation du support et à l’organisation des idées. Le plus important était de rendre les commandes compréhensibles, pas seulement de les afficher.

J’ai travaillé sur l’explication des résultats de scan :

  • reconnaître une machine active ;
  • comprendre les ports ouverts ;
  • repérer des services comme SSH ou VNC ;
  • expliquer l’intérêt de connaître la version d’un service.

Une autre partie importante concernait les bonnes pratiques : demander l’autorisation, prévenir les administrateurs et utiliser une vitesse de scan raisonnable.

Impact et utilité de la preuve

Le diaporama prouve que j’ai compris les étapes principales d’une découverte réseau. La première partie montre comment repérer les machines actives avec nmap -sn. Les slides suivantes expliquent comment aller plus loin : identifier l’OS avec le fingerprint, afficher les services ouverts avec nmap -sV, puis récupérer les versions des services. Cette preuve est utile car elle montre une démarche complète : partir d’un réseau inconnu, trouver les machines, comprendre ce qui tourne dessus et garder en tête les limites légales et techniques du scan.

Auto-évaluation

Cette SAÉ m’a aidé à mieux comprendre le début d’une analyse réseau. Avant, je voyais Nmap surtout comme une commande à taper. Maintenant, je comprends mieux la logique : on commence par découvrir les machines, puis on affine avec les ports, les services et les versions. J’ai aussi compris que la partie sécurité ne se limite pas à trouver des informations : il faut respecter un cadre, demander l’autorisation et éviter de perturber le réseau. Je dois encore progresser sur l’interprétation complète des résultats, mais cette SAÉ m’a donné une première méthode claire.

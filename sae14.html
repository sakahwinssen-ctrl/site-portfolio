SAÉ 14

Se présenter sur Internet

Cette SAÉ m’a permis de créer un site web complet en HTML/CSS pour présenter mon profil, mes compétences et mon parcours.

Retour aux SAÉ Voir mon profil GitHub

Présentation de la SAÉ

Le projet était individuel. Il fallait créer un site web complet en respectant un cahier des charges : structure HTML propre, mise en forme CSS, responsive design, navigation claire, validation W3C et prise en compte de l’accessibilité.

Mes tâches et mes preuves

J’ai codé l’ensemble du site moi-même. J’ai construit la structure HTML, travaillé le CSS, organisé les différentes pages et rendu le contenu consultable en ligne grâce à GitHub Pages. Pour montrer le résultat, j’ai mis le lien direct vers le site publié. Pour montrer la partie technique, j’ai aussi ajouté une capture d’un extrait de mon code HTML.

Accéder directement au site Voir mon profil GitHub

Impact et utilité de la preuve

Cette capture montre la structure HTML de mon site. On y voit les bases d’une page web : le DOCTYPE, la langue de la page, les balises de l’en-tête, le lien vers le fichier CSS, puis l’organisation du contenu avec une partie nav pour la navigation et une partie main pour le contenu principal. Cette preuve est importante parce qu’elle montre que je n’ai pas seulement travaillé l’apparence du site : j’ai aussi construit une base propre et organisée. C’est ce type de structure qui rend un site plus facile à comprendre, à corriger et à faire évoluer.

Auto-évaluation

La principale difficulté a été de rendre le site propre techniquement, notamment pour la validation et l’accessibilité. J’ai dû corriger plusieurs détails comme des balises mal fermées ou des descriptions manquantes. Cette SAÉ m’a fait comprendre qu’un site professionnel doit être propre, lisible et bien structuré, pas seulement agréable visuellement.
